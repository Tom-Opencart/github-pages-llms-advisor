# LLMS Setup Advisor

Статический помощник для настройки модуля `LLMS.txt Generator` под OpenCart и другие магазины с похожей структурой.

- https://tom-opencart.github.io/github-pages-llms-advisor/

## Кратко

Пользователь вставляет URL сайта и получает стартовый набор рекомендаций для настройки `llms.txt` через модуль:

- название сайта;
- краткое описание;
- AI-профиль;
- sitemap;
- официальные источники данных;
- приоритетные разделы;
- FAQ для LLM;
- логика URL;
- идеи для кастомных блоков;
- готовые текстовые поля для копирования в модуль.

## Для чего нужен этот проект

Основная идея проста: владельцу магазина часто трудно не сгенерировать `llms.txt`, а понять, **как именно заполнить модуль под конкретный сайт**.

`LLMS Setup Advisor` помогает сделать первую осмысленную настройку без лишней ручной работы. Это не магический генератор идеального результата, а спокойный стартовый помощник.

## Почему проект сделан именно так

Инструмент изначально подготовлен под `GitHub Pages`, поэтому он:

- не требует сборки;
- не требует backend;
- работает как чистый `HTML + CSS + JavaScript` проект;
- может быть опубликован как обычный статический репозиторий.

## Как работает анализ

Так как `GitHub Pages` не даёт серверной логики, анализ устроен в несколько этапов:

1. прямое чтение сайта из браузера;
2. fallback через публичный текстовый прокси;
3. если сайт всё равно плохо читается, инструмент строит полуавтоматический шаблон по URL и дополнительным подсказкам пользователя.

Из-за этого результат нужно воспринимать как **стартовую конфигурацию**, а не как финальную истину без проверки.

## Что уже умеет MVP

- анализировать главную страницу сайта;
- искать `robots.txt`, `sitemap.xml` и дополнительные sitemap;
- находить сервисные страницы;
- находить новости, блог и обзоры;
- выявлять сигналы B2B, e-commerce и маркетплейсов;
- собирать рекомендации именно под поля `LLMS.txt Generator`;
- формировать черновой preview будущего `llms.txt`;
- давать копируемые блоки для вставки в модуль;
- скачивать JSON-файл со стартовой конфигурацией для будущего импорта в модуль.

## Структура проекта

- `index.html` — интерфейс мастера
- `styles.css` — оформление
- `app.js` — вся клиентская логика анализа и генерации
- `repository-about.txt` — короткие тексты для GitHub repository About
- `.nojekyll` — отключение Jekyll на GitHub Pages
- `.gitignore` — базовая очистка служебных файлов

## Публикация на GitHub Pages

1. создайте отдельный репозиторий;
2. загрузите содержимое этой папки в корень репозитория;
3. включите `GitHub Pages` для branch `main`;
4. откройте опубликованный URL и протестируйте анализ на нескольких сайтах.
